Tool Chain installation

The Broadcom® Tool Chain provides compiler tools and libraries for the development of OpenNSL applications on Broadcom reference platforms. Consult your Broadcom® support representative to determine the correct tool chain to use for your platform, and the correct CPU within that toolchain. For the Interface Masters x86 platform the Ubuntu supplied compiler should be used.

The Broadcom Linux Toolchain is supported on the following systems:

  • Centos 5.2 or later (64-bit)
  • Red Hat Enterprise Linux 5 (64-bit)

Broadcom Linux Tool Chain

The Broadcom Linux toolchain tar ball contains a number of different tool chains for different CPU architectures including GTO, e500v1 and Keystone. The tool chain is available on DocSafe.

The tool chain is supplied as a compressed tar ball with a filename such as fp_7_toolchain.tar.bz2. The tools MUST be installed as follows:

  unix% su
  # cd /
  # rm -rf /projects/nwsoft-toolchains/brl
  # tar jxf <path>/fp_7_toolchain.tar.bz2

Updating the CROSS_COMPILE variable

The OpenNSL compile environment requires that the CROSS_COMPILE variable be set to the full path of the cross compiler you are using. For example, for the GTO cross compiler this should be set to /projects/nwsoft-toolchains/brl_2.0/gto/bin/powerpc-broadcom-linux-gnu- This can be done in one of two ways:

  • Issue the command below. This must be done once for each shell instance:
  unix% export CROSS_COMPILE=/projects/nwsoft-toolchains/brl_2.0/gto/bin/powerpc-broadcom-linux-gnu-
  • Edit the file output/<platform>/Makefile in the OpenNSL and modify the lines that sets the CROSS_COMPILE variable, or add your own line. This need only be done once for all shell instances.

You are now ready to build your OpenNSL application.
