Spanning Tree Groups

For devices that support multiple spanning tree groups (STGs), the STG API functions allow manipulation of STG configuration. For devices not supporting multiple STGs, a single group is assumed to exist (STG 1). After initialization, all VLANS belong to the default spanning tree group (STG 1).

Many of the following functions accept an stp_state argument or return value.

Spanning Tree Port States

  • OPENNSL_STG_STP_DISABLE Disabled
  • OPENNSL_STG_STP_BLOCK Accepts BPDU, does not learn
  • OPENNSL_STG_STP_LISTEN Accepts BPDU, does not learn
  • OPENNSL_STG_STP_LEARN Accepts BPDU and learns, no forwarding
  • OPENNSL_STG_STP_FORWARD Accepts BPDU, normal switching operation
