OpenNSL Directory Structure

The OpenNSL source code is organized in directories as below:

  • documentation - Contains all Doxygen generated documentation
  • driver - Contains all OpenNSL driver initialization files
  • include - Contains all OpenNSL API header files
  • os - Contains platform specific initialization code, root file system and kernel dependencies
  • output - Contains object files generated during build process
  • sdk - Contains Broadcom SDK source code
  • src - Contains the OpenNSL shim layer for Broadcom SDK
  • tools - Contains MAKE targets and tools required to build OpenNSL

The system initializes in the following sequence:

  • SDK: The SDK is initialized with a default configuration for the chip.
  • Platform: The platform's ID, description and physical port mapping information is read from the platform specific structures.
